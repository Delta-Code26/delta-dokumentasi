    Konsep OOP (Object-Oriented Programming) di Java

    Java = Bahasa pemrograman berorientasi objek. Artinya: kamu ngoding sambil mikir kayak bangun dunia nyata. Ada objek, properti, dan perilaku 🧠🚀

    A. Apa itu OOP?

    OOP (Object-Oriented Programming) adalah paradigma pemrograman yang berfokus pada objek.
    Objek = representasi dari sesuatu di dunia nyata (misal: Mobil, Mahasiswa, Pegawai).

    B. 4 Pilar Utama OOP

    Pilar Penjelasan
    Encapsulation Menyembunyikan detail internal objek (pakai private dan getter/setter)
    Inheritance Pewarisan sifat dari kelas induk ke anak (pakai extends)
    Polymorphism Objek punya banyak bentuk (method override, overloading)
    Abstraction Menyembunyikan kompleksitas, hanya menampilkan fitur penting

    C. Class dan Object

    1. Membuat Kelas

    public class Mobil {
    String merk;
    int tahun;

    void nyalakan() {
        System.out.println("Mobil dinyalakan...");
    }
}

    2. Membuat Objek

    public class Main {
    public static void main(String[] args) {
        Mobil m = new Mobil();
        m.merk = "Toyota";
        m.tahun = 2020;
        m.nyalakan();
    }
}

    D. Constructor

    Method khusus yang dipanggil saat objek dibuat.

    public class Mahasiswa {
    String nama;

    // constructor
    Mahasiswa(String namaBaru) {
        nama = namaBaru;
    }
}

    E. Encapsulation

    public class User {
    private String password;

    public void setPassword(String pass) {
        password = pass;
    }

    public String getPassword() {
        return password;
    }
}

    F. Inheritance (Pewarisan)

    class Hewan {
    void suara() {
        System.out.println("Suara hewan");
    }
}

class Kucing extends Hewan {
    void meong() {
        System.out.println("Meong!");
    }
}

    G. Polymorphism

    1. Overriding (Method diganti di child class)

    class Hewan {
    void suara() {
        System.out.println("Suara hewan umum");
    }
}

class Anjing extends Hewan {
    @Override
    void suara() {
        System.out.println("Guk guk!");
    }
}

    2. Overloading (Method sama tapi parameter beda)

    class Calculator {
    int tambah(int a, int b) {
        return a + b;
    }

    double tambah(double a, double b) {
        return a + b;
    }
}

    H. Abstraction (Abstraksi)

    abstract class Kendaraan {
    abstract void jalan();
}

class Motor extends Kendaraan {
    void jalan() {
        System.out.println("Motor jalan di aspal");
    }
}

    🎯 OOP bikin program kamu lebih rapi, reusable, dan scalable. Intinya: kode kamu jadi punya "otak" sendiri 🤖
