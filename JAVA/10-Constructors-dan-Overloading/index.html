    Constructors dan Overloading di Java

    A. Apa itu Constructor?

    Constructor adalah method spesial yang otomatis dipanggil saat objek dibuat. Fungsinya? Buat inisialisasi data atau setup awal objek.

    Ciri khas constructor: - Nama sama dengan nama class - Tidak punya return type - Bisa dibuat lebih dari satu (overloading)

    B. Contoh Constructor Sederhana

    public class Mahasiswa {
    String nama;

    // Constructor
    public Mahasiswa(String namaInput) {
        nama = namaInput;
    }

    void sapa() {
        System.out.println("Halo, saya " + nama);
    }
}

public class Main {
    public static void main(String[] args) {
        Mahasiswa m = new Mahasiswa("Asep");
        m.sapa(); // Output: Halo, saya Asep
    }
}

    C. Constructor Default

    Kalau kamu gak bikin constructor, Java bikin otomatis constructor kosong kayak gini:

    public class Contoh {
    Contoh() {
        // constructor default dari Java
    }
}

    Tapi begitu kamu bikin constructor sendiri, constructor default gak akan ada lagi kecuali kamu bikin manual.

    D. Constructor Overloading

    Kita bisa punya beberapa constructor dengan jumlah/tipe parameter berbeda.

    public class Pegawai {
    String nama;
    int umur;

    // Constructor 1
    public Pegawai(String nama) {
        this.nama = nama;
    }

    // Constructor 2
    public Pegawai(String nama, int umur) {
        this.nama = nama;
        this.umur = umur;
    }

    void info() {
        System.out.println(nama + " - " + umur + " tahun");
    }
}

    E. Keyword this

    • Menunjuk pada objek saat ini
    • Biasanya dipakai buat bedain antara parameter dan atribut
    this.nama = nama; // atribut = parameter

    F. Constructor Chaining (Panggil constructor lain dalam constructor)

    public class Buku {
    String judul;
    int halaman;

    public Buku() {
        this("Tanpa Judul", 0);
    }

    public Buku(String judul, int halaman) {
        this.judul = judul;
        this.halaman = halaman;
    }
}

    Constructor bikin objek kamu lahir langsung dengan identitas. Overloading bikin fleksibel cara lahirnya
